# Benchmark 模式库

本文档提供 AI 产品 Benchmark 设计的常用模式和示例。

---

## Benchmark 设计原则

### 1. 可量化
每个维度必须有明确的测量方法和阈值。

### 2. 场景化
不同场景使用不同的评测维度组合。

### 3. 可对比
支持版本间、竞品间的横向对比。

---

## 常用维度库

### 准确性 (Accuracy)

| 指标 | 定义 | 测量方法 | 典型阈值 |
|------|------|----------|----------|
| 事实正确率 | 答案中事实陈述的正确比例 | 人工标注 / 知识库比对 | ≥ 90% |
| 引用准确率 | 引用来源与内容匹配的比例 | URL 有效性 + 内容相关性检查 | ≥ 95% |
| 代码正确率 | 生成的代码可运行的比例 | 单元测试执行 | ≥ 85% |

### 有用性 (Helpfulness)

| 指标 | 定义 | 测量方法 | 典型阈值 |
|------|------|----------|----------|
| 问题解决率 | 用户问题被解决的比例 | 用户反馈 / 任务完成率 | ≥ 80% |
| 用户满意度 | 用户对回答的满意程度 | Likert 量表（1-5） | ≥ 4.0 |
| 追问率 | 用户需要追问的比例 | 对话轮次统计 | ≤ 20% |

### 安全性 (Safety)

| 指标 | 定义 | 测量方法 | 典型阈值 |
|------|------|----------|----------|
| 有害内容率 | 输出有害内容的比例 | 安全分类器 | ≤ 0.1% |
| 偏见输出率 | 输出存在偏见内容的比例 | 公平性检测器 | ≤ 1% |
| 隐私泄露率 | 输出敏感信息的比例 | PII 检测器 | ≤ 0% |

### 效率 (Efficiency)

| 指标 | 定义 | 测量方法 | 典型阈值 |
|------|------|----------|----------|
| 首字延迟 (TTFT) | 首个 token 输出时间 | 时间戳差值 | ≤ 500ms |
| 生成速度 | tokens/秒 | 统计计算 | ≥ 30 t/s |
| P95 延迟 | 95% 请求的响应时间 | 分布统计 | ≤ 3s |

## 场景化 Benchmark 模板

### 场景 1：AI 问答助手

```json
{
  "product_type": "AI 问答助手",
  "benchmark_set": [
    {
      "dimension": "准确性",
      "weight": 0.4,
      "metrics": [
        {"name": "事实正确率", "threshold": "≥ 90%"},
        {"name": "引用准确率", "threshold": "≥ 95%"}
      ]
    },
    {
      "dimension": "有用性",
      "weight": 0.3,
      "metrics": [
        {"name": "问题解决率", "threshold": "≥ 80%"},
        {"name": "用户满意度", "threshold": "≥ 4.0"}
      ]
    },
    {
      "dimension": "安全性",
      "weight": 0.2,
      "metrics": [
        {"name": "有害内容率", "threshold": "≤ 0.1%"}
      ]
    },
    {
      "dimension": "效率",
      "weight": 0.1,
      "metrics": [
        {"name": "首字延迟", "threshold": "≤ 500ms"}
      ]
    }
  ]
}
```

## Benchmark 设计流程

1. **定义产品类型** → 选择基础维度
2. **分析使用场景** → 调整维度权重
3. **确定测量方法** → 选择具体指标
4. **设定阈值** → 参考行业标准
5. **持续迭代** → 根据数据调整
