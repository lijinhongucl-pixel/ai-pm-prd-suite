# 内部 Skill：幻觉应对 (hallucination-skill)

> 此 Skill 仅供 Sub Agent 内部调用，不直接暴露给用户

---

## 功能

根据用户输入，规划幻觉应对方案。

---

## 输入格式

```json
{
  "product": "AI 客服助手",
  "tolerance": {
    "fact_error": "zero",
    "logic_error": "low",
    "over_extend": "medium"
  },
  "prevention": [
    {"method": "引用溯源", "priority": "高"},
    {"method": "回答限制", "priority": "高"}
  ],
  "detection": [
    {"method": "引用链接检查", "enabled": true},
    {"method": "用户反馈", "enabled": true}
  ],
  "remedy": [
    {"step": "立即响应", "action": "显示'可能有误'"},
    {"step": "提供修正", "action": "正确答案+原因"},
    {"step": "用户安抚", "action": "道歉+补偿（如有）"}
  ],
  "responsibility": {
    "ai_provided_wrong_info": "平台责任",
    "user_did_not_verify": "共同责任"
  }
}
```

---

## 输出格式

```markdown
# 幻觉应对策略

## 产品：AI 客服助手

---

## 幻觉容忍度

| 类型 | 描述 | 容忍度 | 原因 |
|------|------|--------|------|
| 事实错误 | 编造不存在的信息 | 零容忍 | 可能误导用户 |
| 逻辑错误 | 推理链条有漏洞 | 低容忍 | 需快速修复 |
| 过度延伸 | 超出问题范围回答 | 可接受 | 危害较小 |

---

## 预防机制

### 高优先级

1. **引用溯源**
   - 要求：每个事实性陈述必须有来源
   - 格式："根据 [文档名]..."
   - 无法溯源时：明确告知"我无法找到确切来源"

2. **回答限制**
   - 原则：只回答知识库内的问题
   - 超出范围：告知用户"这个问题超出了我的知识范围"

### 中优先级

3. **不确定性声明**
   - 适用：置信度 < 80% 的回答
   - 格式："根据现有信息，可能是...，但建议您核实"

---

## 检测机制

| 方式 | 是否启用 | 说明 |
|------|----------|------|
| 引用链接检查 | ✅ | 检查引用是否存在 |
| 用户反馈 | ✅ | 用户可标记"回答有误" |
| 知识库匹配 | ⚪ 可选 | 检查答案是否在知识库 |

---

## 补救方案

发现幻觉后的处理流程：

1. **立即响应**：显示"刚才的回答可能有误，正在核实..."
2. **提供修正**：显示正确答案 + 错误原因
3. **用户安抚**：道歉 + 补偿（如有损失）

---

## 责任边界

| 场景 | 责任归属 |
|------|----------|
| AI 提供错误信息，用户未核实 | 共同责任 |
| AI 提供错误信息，造成直接损失 | 平台责任 |
| AI 明确声明不确定，用户仍采纳 | 用户责任 |

---

## PM 决策确认

- 事实错误：零容忍
- 预防机制：引用溯源 + 回答限制
- 发现后：立即告知 + 提供修正
```

---

## 使用示例

```
教练：让我整理幻觉应对方案...

[内部调用 hallucination-skill]

教练：整理如下：
      - 事实错误：零容忍
      - 预防：引用溯源 + 回答限制
      - 补救：立即告知 + 提供修正
      
      你确认吗？
```
